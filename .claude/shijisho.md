# 指示書

完了した項目にはチェックを付けてください。矛盾点があれば必ず質問してから進めてください。

## 概要

このサイトでは、駅名標を「手動で生成」または「路線を作って生成」することができます。すべての作業はローカルで完結し、アカウント連携機能などは実装しません。生成したSQLiteファイルは、ユーザーが必要に応じてダウンロードすることができます。

## 実装13：路線図の乗り換え表示（v2026.3.0-beta17アップデート内容）

現在の路線図モードにおける乗り換え案内の出力は、かなりわかりにくいものになっています。これを以下の指示に沿って作り変えてください。

1. [ ] 現在の路線乗り換え表示表記（駅名の横に路線の色と白文字）は廃止してください。
2. [ ] 乗り換え路線は、初期状態では一つも選択しないようにしてください。
3. [ ] 選択した種別すべてが通過する駅の場合、その駅の乗換情報を表示するかどうかは選択制にしてください。
4. [ ] 縦方向モードの場合は、以下のように実装してください。
   1. [ ] 駅名の側が右の場合、駅名（primaryとsecondaryのいずれか長い方）の左上を基準として、路線バッジ（駅番号バッジではない）を表示し、その右に路線名を書く。最大3行となるようにし、3行を超える場合は3行のうち最も長いものの右端を基準として右に積み同様に繰り返す
   2. [ ] 駅名の側が左の場合、左が路線バッジで右が路線名となること以外はすべて左右を反転して同様に実装する。
5. [ ] 横方向モードの場合は、以下のように実装してください。
   1. [ ] 駅名スタイルが上（縦書き）の場合、駅名（primaryとsecondaryのいずれか長い方）の左上を基準として、路線バッジ（駅番号バッジではない）を表示し、その上に縦書きで路線名を書く。最大3行となるようにし、3行を超える場合は3行のうち最も長いものの上端を基準として上に積み同様に繰り返す
   2. [ ] 駅名スタイルが下（縦書き）の場合、すべて上下を反転して同様に実装する。

## 実装14：直通運転と支線

1. [x] 現在のデータベース構造では、「直通運転路線」が実装できません。例えば、「東京メトロ丸ノ内線」は、M06中野坂上駅は支線においてもM06でありMb06ではありません（詳しくは「参考資料」の「丸ノ内線」と「丸ノ内線（方南町方面）」を参考にしてください）。修正を計画してください。

## 実装15：地下鉄丸ノ内線実装（v2026.3.0-beta18アップデート内容）

1. [ ] 現在のデータ構造では、地下鉄丸ノ内線方南町支線を追加したいときに、地下鉄丸ノ内線と共有する駅「中野坂上駅」をMbではなくMとして扱うことができません。今後も路線作成時に支線や分岐線などが複数できることを念頭に置きながら、何か解決法を提案してください。
2. [ ] 上記の問題が解決したら、サンプルデータベースに、このファイル末尾の「参考資料」を読みながら、地下鉄丸ノ内線を追加してください。

## 実装16：不具合修正

1. [ ] シンプル入力にはすでに解決策を埋め込んでありますが、一部画像生成時にフォントの読み込みが完了しておらず、文字がズレた状態で入ってしまう不具合があります。canvasの仕様に注意をはらいながら、ローディング画面が閉じたあとは確実にフォントがcanvas内で描画されるように対策方法を検討してください。
